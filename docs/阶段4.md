## 阶段四：跨端启动与内网穿透真机预览

**目标：** 启动 Web 和 App 服务，并使用 tunnel 让手机直接扫码。

🤖 **请发给你的 AI Agent：**

> UI 骨架已搭好。现在我们要测试多端运行。
> 
> 1. 请帮我修改根目录的 `package.json`，在 `scripts` 中添加一个专门用于云端调试 App 的命令：
>    `"dev:mobile": "turbo run dev --filter=native -- --tunnel"`
>    *(注：如果是 `apps/expo` 请替换 filter 的名字)*
> 2. 请告诉我，如果在终端执行 `npm run dev:mobile`，系统会发生什么？
>    **注意**：当前 SDK 推荐使用本地 CLI，通过 `npx expo` 运行而不是全局 `expo-cli`。
>    如果需要清缓存，可先进入 `apps/native` 并运行
>    ````bash
>    EXPO_ROUTER_APP_ROOT=./app npx expo start --tunnel --clear
>    ````
>    拿到二维码后，用手机上的 Expo Go App 扫描即可预览 5 Tab 界面。

> 3. 为了避免 `babel.config.js` 中缺少 `expo-router/babel` 插件而导致的
>    `Invalid call at line 2: process.env.EXPO_ROUTER_APP_ROOT` 错误，
>    请确保 `apps/native/babel.config.js` 中包含该插件，并且脚本
>    启动前通过环境变量 `EXPO_ROUTER_APP_ROOT=./app` 指定路由根目录。

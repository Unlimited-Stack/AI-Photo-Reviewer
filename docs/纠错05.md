在当前的 Expo SDK 54 中，Expo Router 已经原生接管了路由解析。强行注入绝对路径反而破坏了 Metro 最新的 `collect-dependencies` 静态分析机制，并导致了严重的缓存污染。

## ✅ 现在的修复任务（大道至简）：
请帮我彻底移除旧的 Hack 补丁，恢复最纯净的官方 Monorepo 配置。请按顺序严格执行以下 4 步：

### 第一步：删除错误的 Babel 插件
请直接删除文件：`apps/native/babel-plugin-inline-router-root.js`（如果存在的话）,或者与之相关的Babel插件。

### 第二步：重置 Babel 配置
请将 `apps/native/babel.config.js` 的内容完全替换为以下极简代码，剔除任何与 Router 相关的自定义 plugin，请不要再定义别的旧的或已淘汰的插件：

```javascript
module.exports = function (api) {
  api.cache(true);
  return {
    presets: ['babel-preset-expo']
  };
};
```

### 第三步：配置 Monorepo 原生 Metro 解析
请将 `apps/native/metro.config.js` 的内容完全替换为以下代码。这是新版 Expo 识别 Turborepo 根目录和路由体系的核心：

```javascript
const { getDefaultConfig } = require('expo/metro-config');
const path = require('path');

// 找到 native 项目目录和 Monorepo 根目录
const projectRoot = __dirname;
const workspaceRoot = path.resolve(projectRoot, '../..');

const config = getDefaultConfig(projectRoot);

// 1. 监控 Monorepo 根目录，使得 UI 包和全局 node_modules 能被识别
config.watchFolders = [workspaceRoot];

// 2. 让 Metro 优先在 native 目录下找依赖，找不到再去根目录找
config.resolver.nodeModulesPaths = [
  path.resolve(projectRoot, 'node_modules'),
  path.resolve(workspaceRoot, 'node_modules'),
];

// 3. 确保 Expo Router 在 Monorepo 中正常工作
config.resolver.disableHierarchicalLookup = true;

module.exports = config;
```

### 第四步：执行核弹级缓存清理

```bash

```
